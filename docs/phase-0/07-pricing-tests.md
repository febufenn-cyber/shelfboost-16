# Pricing Tests

## Rule

Do not ask only whether a participant “would pay.” Present a concrete next batch, scope, price, and start condition.

## Models to test

### A — paid batch

Best when the job is infrequent or the buyer wants low commitment.

- fixed number of audited products;
- defined number of draft fields;
- one review cycle;
- export-only delivery.

### B — monthly store plan

Best when launches, refreshes, and monitoring recur.

- active-product band;
- monthly optimization allowance;
- recurring catalog scan;
- controlled review workflow.

### C — agency portfolio

Best when one operator manages several stores.

- store band;
- client-specific brand profiles;
- reviewer seats;
- usage and client export boundaries.

### D — audit plus ongoing monitoring

- paid initial catalog diagnosis;
- recurring fee for new-product and deterioration alerts;
- separate batch fee for remediation.

## Price interview

1. What does the current workflow cost in hours and external spend?
2. What exact scope should the next paid engagement include?
3. Which unit is easiest to approve: product, batch, store, or month?
4. At what price does the participant need another approver?
5. What outcome or safeguard must be written into the pilot?
6. Ask for payment, deposit, or signed commitment.

## Evidence coding

- `paid`
- `deposit_committed`
- `contract_committed`
- `budget_owner_review`
- `verbal_yes_only`
- `price_objection`
- `value_objection`
- `timing_objection`
- `trust_objection`
- `no_budget`

Do not count `verbal_yes_only` as commercial validation.
